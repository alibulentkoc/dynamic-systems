State-Space Modeling

Back in Lesson 3 we said a system's state is the minimum you must know now to predict its future — position and velocity for a spring. State-space modeling turns that idea into the language of modern control: instead of one high-order equation, a tidy set of first-order ones, written as matrices. It's how robots, spacecraft, and self-driving cars are actually controlled.

The one idea to hold onto

Track the state, one first-order equation each

A transfer function hides the system's insides and only relates input to output. State-space does the opposite: it names every internal state variable (one per energy store) and writes a simple first-order equation for how each one changes. Stack them into a vector x and the whole system — however big — becomes two matrix equations:

ẋ = A x + B u   // how the state evolves
y = C x + D u   // what we measure

Each matrix has a plain-English job: A — internal dynamics: how the state pushes itself B — how the input u drives the state C — how outputs are read off the state D — any direct input-to-output path. The star is A: it holds all the system's natural behavior.

The mass–spring–damper, in state-space

Pick the two states everyone already knows: position x₁ and velocity x₂. The first equation is just the definition of velocity; the second is Newton's law rearranged (from Lesson 2):

ẋ₁ = x₂   // velocity is the rate of position
ẋ₂ = −(k/m)·x₁ − (c/m)·x₂ + (1/m)·F   // Newton, solved for acceleration

Written as matrices, that's A = [[0, 1], [−k/m, −c/m]], B = [[0], [1/m]], C = [1, 0] (we measure position), D = 0. Two first-order equations replace one second-order one — and that trade scales to any size.

Interactive · the anatomy of A

Its eigenvalues are the poles

Here's the unification that ties this whole course together. The eigenvalues of the matrix A — the special numbers λ for which A·v = λ·v — are exactly the poles of the transfer function, which are exactly the roots λ you found by guessing eλt in Lesson 3. Change m, c, k and watch A's entries update, its eigenvalues move in the s-plane, and the impulse response change with them.

So state-space, transfer functions, and the raw differential equation are three views of one thing. The denominator's roots, the poles, the λ's, and the eigenvalues of A are all the same numbers — the system's natural frequencies. State-space just stores them inside a matrix instead of a polynomial.

Interactive · the state as a flow

A defines a field; eigenvectors are its natural directions

The equation ẋ = Ax says: at every point in state space, the matrix A tells the state which way to move. That's a flow field — the same phase portrait from Lesson 3, now revealed as the action of A. When A's eigenvalues are real (over-damped), there are special straight-line directions — the eigenvectors — along which the state simply decays without curving. When the eigenvalues are complex (under-damped), no such lines exist and the state spirals.

Why engineers reach for state-space

Many inputs & outputs (MIMO) — trivial Nonlinear & time-varying systems Kalman filters & state estimation Optimal & model-predictive control Robotics, aerospace, autonomy Clean for computers (first-order ODEs)

A transfer function is one-input, one-output; the moment a system has several actuators and sensors — a drone with four rotors, a robot arm with six joints — state-space handles it in the same tidy form, just with bigger matrices. It also opens the door to estimating hidden states you can't measure directly (the Kalman filter) and to optimal controllers that weigh performance against effort. Nearly all modern, computer-based control lives here.

Key takeaways

  • State-space describes a system by its state — one first-order equation per energy store — written compactly as ẋ = Ax + Bu, y = Cx + Du.
  • The matrices have plain jobs: A = internal dynamics, B = input coupling, C = output map, D = feedthrough.
  • The mass–spring–damper becomes a 2×2 system with states (position, velocity) — two first-order equations replacing one second-order one.
  • Eigenvalues of A = poles = the λ roots. The same natural frequencies, stored in a matrix instead of a polynomial.
  • A defines a flow on state space; real eigenvalues give straight-line eigenvector directions, complex ones give spirals.
  • State-space scales to many inputs and outputs and underpins Kalman filtering, optimal control, robotics, and autonomy.

Quick check

How many state variables does a mass–spring–damper need, and what are they?

Two: position and velocity. That's one per energy store (spring's potential, mass's kinetic), which is also the system order and the number of initial conditions — all the same count.

What is the relationship between the eigenvalues of A and the system's poles?

They are identical. The eigenvalues of A are the roots of its characteristic equation, which is the same polynomial whose roots are the transfer function's poles. State-space and transfer-function views agree exactly on the natural frequencies.

Why prefer state-space for a drone with four rotors and many sensors?

Because it's naturally multi-input, multi-output. A transfer function relates one input to one output; state-space packs all the actuators into B, all the sensors into C, and keeps the same tidy form — plus it supports state estimation and optimal control that MIMO systems need.

Exercises

  1. Calculation. Write the state-space matrices A and B for a mass–spring–damper with m = 2, k = 8, c = 4, using states (position, velocity).
  2. Calculation. For that A, the eigenvalues satisfy λ² − (trace)λ + det = 0. Compute the trace and determinant of A and find the eigenvalues. Do they oscillate?
  3. Calculation. A first-order system τẋ + x = u has a single state. Write it as ẋ = Ax + Bu (find the scalars A and B) and give its one eigenvalue.
  4. Design. You have a system whose A eigenvalues sit too close to the imaginary axis (slow, ringing). In state-feedback control you replace u with u = −K·x, which changes A into A − BK and thus moves the eigenvalues. Explain in words what you're trying to achieve by choosing K, and how it connects to everything you learned about pole locations.
Show worked solutions

1. A = [[0, 1], [−k/m, −c/m]] = [[0, 1], [−4, −2]] (since k/m = 4, c/m = 2). B = [[0], [1/m]] = [[0], [0.5]]. C = [1, 0], D = 0.

2. trace(A) = 0 + (−c/m) = −2; det(A) = (0)(−c/m) − (1)(−k/m) = k/m = 4. The eigenvalue equation is λ² + 2λ + 4 = 0 → λ = (−2 ± √(4−16))/2 = −1 ± j√3. Complex with negative real part → stable and oscillates (it's underdamped; ζ = c/(2√(km)) = 4/(2√16) = 0.5).

3. Rearrange: ẋ = −(1/τ)x + (1/τ)u, so A = −1/τ, B = 1/τ (scalars, a 1×1 system). Its single eigenvalue is λ = −1/τ — the familiar first-order pole on the negative real axis.

4. Choosing K is choosing where to place the closed-loop eigenvalues. Since the eigenvalues of A − BK are the new poles, and pole location sets speed, damping, and stability (Lessons 4–6), you pick K to move them further left (faster settling) and toward the real axis / better angle (more damping, less ringing) — without pushing so hard that control effort or noise sensitivity explode. This is "pole placement," and it's the same design goal from the s-plane, now achieved by reshaping A directly. The entire course converges here: to control a system is to move its eigenvalues to where you want them.
